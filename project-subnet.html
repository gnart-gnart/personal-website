IPv4 Subnetting & Network Planning CLI (Python)

Why I built this

While actively studying for the CCNA, I quickly realized that doing subnetting math on paper—figuring out network boundaries, broadcast addresses, and valid host ranges—is notoriously tedious. Because my background is in software engineering, my instinct whenever I encounter repetitive math is to write a script to automate it. I wanted a project that would force me to truly understand how IP addressing works at a binary level, rather than just memorizing a cheat sheet for the exam. It was the perfect opportunity to bridge my programming experience with my pivot into network infrastructure.

What I built

I developed a command-line utility in Python that takes an IP address and a CIDR notation (like 192.168.1.50/27) as user input and instantly calculates the network topology. The script returns:

  • The Network Address (the mathematical starting point of the subnet).
  • The Broadcast Address (the highest address in the subnet used for broadcasting).
  • The Subnet Mask in dotted decimal format (e.g., 255.255.255.224).
  • The total number of available hosts, along with the specific usable IP range.

It acts as a lightweight, fast capacity planning tool that can be run from any terminal, which is exactly the kind of automation script a systems administrator would use on a daily basis.

How I did it

The easiest way to build this in Python would be to just import the built-in ipaddress module, which does all the heavy lifting automatically. However, since the goal was to learn, I intentionally chose not to use it. Instead, I built the logic from scratch using bitwise operations.

I wrote functions to convert the dotted-decimal IP address string into a 32-bit binary integer. From there, I used the CIDR prefix to dynamically generate a binary subnet mask. By applying bitwise AND/OR operations between the IP and the mask, I could calculate the exact network and broadcast boundaries mathematically. I also implemented input validation to catch edge cases, like ensuring users couldn't input a /32 mask and expect a usable host range, or typing impossible octets (like 300.1.1.1).

What I learned

This project solidified my understanding of IPv4 architecture better than any textbook could. Forcing the code to handle bitwise logic made the concept of a "subnet mask" fundamentally click—it's no longer just a string of numbers to me, but an actual binary filter. Furthermore, it demonstrated the value of automation in infrastructure. Understanding how to use Python to interact with and calculate network data is incredibly powerful, and it proved to me that my software development skills are highly transferable to managing and provisioning enterprise networks.
